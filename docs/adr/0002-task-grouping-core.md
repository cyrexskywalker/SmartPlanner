# ADR 0002: Core-логика группировки задач

## Контекст
Список задач группирует элементы по приоритету и поддерживает сортировку в двух направлениях. Это бизнес-логика, поэтому ее нужно проверять без Android UI-классов.

## Решение
Группировка задач на Android вынесена в `TaskGroupingService`. Он получает обычные доменные модели и возвращает `TaskGroup`. Репозиторий хранит задачи и делегирует группировку этому сервису, а преобразование групп в `TaskListItem` остается на уровне UI.

## Последствия
Правила сортировки покрыты unit-тестами. Фрагмент и адаптер не содержат алгоритм группировки, а data-слой больше не зависит от UI-моделей.
