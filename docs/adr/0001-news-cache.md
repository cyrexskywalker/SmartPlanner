# ADR 0001: Политика кэширования новостей

## Контекст
Экран новостей не должен зависеть только от живого сетевого запроса. Пользователь должен быстро увидеть уже загруженные новости, а приложение должно нормально переживать временные проблемы с сетью.

## Решение
Загрузка новостей проходит через репозиторий. Экран сначала получает сохраненные статьи из кэша, а затем запускает обновление из сети. Если сетевой запрос успешный, свежие данные заменяют старый кэш.

Список новостей хранится 24 часа. Если кэш устарел или поврежден, он очищается и не используется.

На Android изображения кэшируются отдельно: в памяти для быстрых повторных показов во время текущего запуска и на диске для повторного использования после перезапуска приложения.

## Последствия
UI не знает, откуда пришли данные: из сети или из локального хранилища. Кэш остается простым, предсказуемым и легко очищается.
